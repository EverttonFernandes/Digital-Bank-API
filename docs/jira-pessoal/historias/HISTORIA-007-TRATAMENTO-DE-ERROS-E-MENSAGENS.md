# HISTORIA-007 — Tratamento de Erros e Mensagens Padronizadas

## Tipo

- não funcional

## Objetivo da História

Padronizar respostas de erro e mensagens da API para refletirem corretamente o que aconteceu.

## Valor de Negócio

Esta história melhora a clareza para quem consome a API e fortalece os testes funcionais.

## Critérios de Aceite

- respostas de erro devem ser consistentes
- mensagens devem possuir `key` e `value`
- testes devem validar a mensagem correta para cada regra de negócio violada
